===ISSUE===
FILE: crypto/rsa/src/rsa_keygen.c
LINE: 258-259
SEVERITY: low
TITLE: Unnecessary memory allocation in RSA_GetPublicExp
PROBLEM: r1 and r2 are allocated with bits size instead of bits>>1. Since p-1 and q-1 (where p and q are roughly bits/2 sized primes) fit in bits/2 + 1 bits, using bits size is wasteful for a bin size optimization PR.
CODE:
```
BN_BigNum *r1 = BN_Create(bits);
BN_BigNum *r2 = BN_Create(bits);
// r1 = p - 1, r2 = q - 1 where p and q are ~bits/2 sized primes
```
FIX:
```
BN_BigNum *r1 = BN_Create(bits >> 1);
BN_BigNum *r2 = BN_Create(bits >> 1);
```
===END===
