Issue: The `-DHITLS_AARCH64_PACIASP` flag was removed from CC_DEFINE_FLAGS. This disables Pointer Authentication (PAC) for all ARMv8 assembly code (AES, SHA, SM2/3/4, ECC, ML-KEM, etc.). PAC is a significant security hardening feature that helps prevent ROP (Return-Oriented Programming) attacks on ARMv.3-A+ processors. This change affects not just the new ML-KEM code but ALL existing ARMv8 assembly implementations.
Fix:
"CC_DEFINE_FLAGS": [
    "-DHITLS_CRYPTO_EAL_REPORT",
    "-DHITLS_CRYPTO_NIST_ECC_ACCELERATE",
    "-DHITLS_CRYPTO_BN_COMBA",
    "-DHITLS_CRYPTO_AES_PRECALC_TABLES",
    "-DHITLS_AARCH64_PACIASP",
    "-DHITLS_CRYPTO_SM2_PRECOMPUTE_512K_TBL"
]
