Based on my thorough review, I've identified several critical issues:

===ISSUE===
FILE: crypto/mldsa/CMakeLists.txt
LINE: 15-28
SEVERITY: high
TITLE: Missing ARMv8 architecture check before including assembly files
PROBLEM: The CMakeLists.txt includes ARMv8 assembly files when HITLS_CRYPTO_MLDSA_ARMV8 is ON, but doesn't verify the build architecture is actually ARMv8. Building with HITLS_CRYPTO_MLDSA_ARMV8=ON on x86_64 will fail to compile or produce incorrect results. The check should also verify HITLS_ASM_ARMV8 is enabled.
CODE:
```cmake
if(HITLS_CRYPTO_MLDSA_ARMV8)
    list(APPEND _mldsa_sources ${CMAKE_CURRENT_SOURCE_DIR}/src/asm/decompose_armv8.S)
    ...
```
FIX:
```cmake
if(HITLS_CRYPTO_MLDSA_ARMV8 AND HITLS_ASM_ARMV8)
    list(APPEND _mldsa_sources ${CMAKE_CURRENT_SOURCE_DIR}/src/asm/decompose_armv8.S)
    ...
```
===END===

===ISSUE===
FILE: cmake/hitls_define_dependencies.cmake
LINE: 420-424
SEVERITY: high
TITLE: Missing HITLS_CRYPTO_MLDSA_ARMV8 dependency definition
PROBLEM: HITLS_CRYPTO_MLDSA_ARMV8 is not defined in hitls_define_dependencies.cmake, unlike all other *_ARMV8 features. This means there's no automatic validation that HITLS_ASM_ARMV8 is enabled when HITLS_CRYPTO_MLDSA_ARMV8 is ON. Other ARMV8 features (like HITLS_CRYPTO_BN_ARMV8, HITLS_CRYPTO_AES_ARMV8) have proper dependency definitions.
CODE:
```cmake
hitls_define_dependency(HITLS_CRYPTO_MLDSA
    DEPS HITLS_CRYPTO_PKEY
        HITLS_CRYPTO_PKEY_SIGN HITLS_CRYPTO_SHA3 HITLS_BSL_PARAMS HITLS_BSL_OBJ_DEFAULT
    CHILDREN HITLS_CRYPTO_MLDSA_CHECK
)
```
FIX:
```cmake
hitls_define_dependency(HITLS_CRYPTO_MLDSA
    DEPS HITLS_CRYPTO_PKEY
        HITLS_CRYPTO_PKEY_SIGN HITLS_CRYPTO_SHA3 HITLS_BSL_PARAMS HITLS_BSL_OBJ_DEFAULT
    CHILDREN HITLS_CRYPTO_MLDSA_CHECK HITLS_CRYPTO_MLDSA_ARMV8
)
hitls_define_dependency(HITLS_CRYPTO_MLDSA_ARMV8 DEPS HITLS_CRYPTO_MLDSA)
```
===END===

Add HITLS_CRYPTO_MLDSA_ARMV8 to the dependency system as described above.
