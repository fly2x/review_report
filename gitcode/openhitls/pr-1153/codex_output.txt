PROBLEM: The scalar reference computes the magnitude with bit-twiddling, which rejects `INT32_MIN`. This replacement uses `ABS` and then a signed `CMGE`. On AArch64, `ABS` does not make `INT32_MIN` positive, so `CMGE` sees it as negative and the bound check can return a false success for that value.

    // n = z[j] - (n & ((uint32_t)z[j] << 1));
    // if (n >= t) return false;
