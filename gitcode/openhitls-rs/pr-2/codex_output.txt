TITLE: CCM mode silently truncates caller-supplied IVs
PROBLEM: For CCM, any IV longer than 12 bytes is silently shortened to its first 12 bytes before encryption and decryption. That collapses distinct nonces onto the same effective IV, which is a cryptographic misuse and can cause nonce reuse under the same key.         let ctx = sha2_256_newctx();
        if ctx.is_null() {
            return Err("CRYPT_SHA2_256_NewCtx failed".to_string());
        }

        let ret = sha2_256_init(ctx);
