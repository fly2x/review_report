SEVERITY: low
TITLE: Ciphertext not cleared after use, potential key material leak
REVIEWERS: CLAUDE
CONFIDENCE: likely
PROBLEM: In engineGenerateSecret(), the shared key is cleared after use but the ciphertext field is not. The ciphertext contains cryptographic material that should be cleared from memory after use.
CODE:
```
if (sharedKey != null) {
    byte[] result = sharedKey;
    sharedKey = null;  // Cleared after use
    return result;
}
```
FIX:
```
if (sharedKey != null) {
    byte[] result = sharedKey;
    sharedKey = null;
    if (ciphertext != null) {
        Arrays.fill(ciphertext, (byte) 0);
        ciphertext = null;
    }
    return result;
}
```

CODE:
```

}

FIX:
```
public FrodoKEMGenParameterSpec(String name) {
    if (name == null) {
        throw new NullPointerException("name cannot be null");
